## Core Rule

**Two grammars. Two contexts only.**

| Context | Grammar | When |
|---------|---------|------|
| Scan | Row | Library list, Inbox list, Casebook list, Product signal list |
| Read | Reading object (raised card) | SignalDetail, TopicDetail evidence, Compare result, Brief output |

**Do not use `surfaceCardStyle` for any list row.** `surfaceCardStyle` is for reading objects only.

**Do not add new grammars.** No tile grid, no masonry, no two-column card gallery.

## Task 1 — Row primitive in `components.tsx`

**File:** `src/ui/components.tsx`

```typescript
/** Row grammar — for scan-page list items (Inbox, Library, Casebook, Product list) */
export function scanRowStyle(extra?: CSSProperties): CSSProperties {
  return {
    display: "flex",
    alignItems: "baseline",
    gap: tokens.spacing.sm,

    background: "transparent",
    transition: "background 140ms ease",
    cursor: "default",

```

**Acceptance:** `surfaceCardStyle` and `scanRowStyle` coexist. No existing component is changed yet.

## Task 2 — Inbox list → row grammar

**File:** `src/ui/InboxView.tsx`

The current inbox renders each signal with a card-like container (using inline `padding`, `borderRadius`, `boxShadow` or `surfaceCardStyle`). Convert each signal list item to use `scanRowStyle`.

**Row anatomy for each signal item:**
```
[left accent bar 3px × 28px, color = mode accent] [body: title (14px, weight 600) + meta line (12px, softInk)] [right: status stamp + date (11px, softInk)]
```

- Left accent: a `4px wide × 28px tall` vertical bar using `--dlens-mode-accent`. This is the only decoration.
- Title: `signal.title` or preview text, 14px, `tokens.color.ink`, weight 600, single line with `lineClamp(1)`.
- Meta line: domain/source + `·` + captured date, 12px, `tokens.color.softInk`.
- Status stamp: reuse existing `<Stamp>` component for `inboxStatus`.
- Date: 11px, `tokens.color.softInk`, right-aligned.

**No card elevation, no `boxShadow`, no `borderRadius` on row.**

Add `data-scan-list` to the list wrapper and `data-scan-row` to each row element.

Inject `SCAN_ROW_HOVER_CSS` once via a `<style>` tag in the component (same pattern as `PRODUCT_SIGNAL_MOTION_CSS` in `ProductSignalViews.tsx`).

**Filter chips (全部 / 未處理 / 已標記 / 已歸檔) stay as-is.** Do not convert them.

**Acceptance:**
- Inbox list items have no box shadow and no raised background.
- Each row has the 3px left accent bar.
- Hover produces a subtle background tint only.
- `npm run typecheck` passes.

## Later Task 7 — Product Batch Export tray

**File:** `src/ui/ProductSignalViews.tsx`

V2.2 introduced a Batch Export concept: user manually selects signals, picks a format, gets a copyable output. Implement the minimum viable version.

**Status:** Later slice. Do not include in V2.3 Core. Start this only after T1–T6 have landed and the Product classification list still feels correct in Chrome.

### State additions (local to the component, not global state):
```typescript
const [exportSel, setExportSel] = useState<Set<string>>(new Set());
const [exportFormat, setExportFormat] = useState<"long" | "short">("long");
const [showExportTray, setShowExportTray] = useState(false);
```

### Selection:
On the signal list page (`classification`), each row gets a checkbox on the far left. Checking it adds to `exportSel`. The checkbox is a native `<input type="checkbox">` with custom styling (no third-party).

### Export tray:
When `exportSel.size > 0`, show a sticky tray at the bottom of the classification page:
```
[n 則已選] [format toggle: 原文優先 / 精簡決策] [複製匯出]
```
- Tray: `position: sticky; bottom: 0`, background `tokens.color.elevated`, `border-top: 1px solid tokens.color.line`, padding `10px 16px`.
- Format toggle: two buttons styled like the existing `SecondaryButton`. Only one active at a time.
- Copy button: uses `navigator.clipboard.writeText()`. On success, shows a brief inline confirmation "已複製" for 1500ms then resets.

### Output format:
- **原文優先 (long):** For each selected signal: headline + verdict/category + experimentHint (if present) + top 2 evidence quotes (if present) + source refs.
- **精簡決策 (short):** For each selected signal: headline + verdict + experimentHint only (no quotes, no agent task).

Separate signals with `\n---\n`.

Do **not** include `agentTaskSpec.taskPrompt` in the MVP batch output. Single-card prompt remains an explicit future action, not the default batch handoff.

**Do not add a new route or page for Export.** The tray is inline on the classification page.

**Acceptance:**
- Selecting signals shows the tray.
- Both formats produce readable text output.
- Copy button works.
- Deselecting all signals hides the tray.
- `npm run typecheck` passes.
