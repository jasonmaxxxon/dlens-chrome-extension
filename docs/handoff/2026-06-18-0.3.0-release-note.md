# DLens 0.3.0 release note

Date: 2026-06-22
Baseline: extension `0.2.1` at tag `v0.2.1`
Release range: `v0.2.1..v0.3.0`

## Why 0.3.0

0.3.0 is the Visual Reset A close: the first user-visible UI overhaul since
0.2.0. It is additive presentation work only. It does not change storage
shape, backend contracts, crawler behavior, ViewModel derivation, classifier
logic, command targets, or signal-packet contracts.

## What shipped

### Popup shell (PR #60)

- Mode switching now uses a segmented control with a sliding thumb.
- Backend reachability and processing state are integrated into the masthead
  status rail.
- Idle utility hints expose `⌘1-3` mode shortcuts and `⌘K` command entry.

### PR Evidence ledger (PR #61)

- Ledger rows use audit-style numbering and denser evidence rhythm.
- Quoted evidence uses the shared italic-serif `QuoteBlock` treatment.
- Existing source-link command surfaces remain unchanged.

### Topic detail audit rhythm (PR #63)

- Topic overview, themes, lanes, and source list moved onto shared
  `SurfaceCard` / `SectionHeader` framing.
- Sage topic accent rails use the mode-accent CSS variable path.
- Source-row command wiring remains unchanged.

### Compare hero billboard (PR #64)

- Compare result opens with a raised hero card and diagonal three-color
  backdrop.
- The hero includes a mode-accent stripe, compact verdict label, and Post A /
  Post B stance cells.
- The hero slice is render-only: cluster dock, casebook, evidence rows,
  processing strip, Compare VM, and async state derivation remain untouched.

### Product action cards (PR #65)

- Product action cards use steel-blue (`#234f7a`) accent rails through
  `--dlens-mode-accent`.
- `ProductReadinessChip` surfaces saved / crawling / ready status inside the
  action card.
- A render-level guard keeps exactly one `tokens.shadow.raised` action surface
  in the isolated card fixture.
- Hover lift is transform-only and does not shift layout.

## Foundation

- PR #58 converged the active design source: `src/ui/tokens.ts` remains the
  single live contract, `design-system/DESIGN.md` is archived, and mockups moved
  under `docs/mockups/references/`.
- PR #59 extracted shared primitives including `SurfaceCard`,
  `SectionHeader`, `EvidenceRow`, `StatusDot`, `StatusRail`, `KeyHint`, and
  `QuoteBlock`.
- PR #62 recorded the post-merge LOC gap: row-level primitive adoption across
  the five large View files is still a separate refactor track.

## Contract notes

- `DLENS_SIGNAL_PACKET_VERSION` remains `v3`.
- No storage migration.
- No backend tag in this release.
- No ViewModel, classifier, command target, storage seam, backend, or content
  script change.
- Mode accent semantics remain unchanged for archive / topic / product /
  pr-evidence. Visual treatment evolved under the Visual Reset A plan.

## Out of scope

- Row-level primitive adoption and LOC reduction across the five large View
  files.
- `LibraryView` surface treatment. It is not a Visual Reset A marquee surface.
- Pipeline Spine `ui.ready` live harness work. It is already locked under
  `TRACE`.
- Version bump files. This note does not edit `package.json`, `wxt.config.ts`,
  `src/ui/version.ts`, or manifest tests.
