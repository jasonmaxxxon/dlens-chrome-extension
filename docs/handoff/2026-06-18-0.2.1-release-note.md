# DLens 0.2.1 release note

Date: 2026-06-18
Baseline: extension `0.2.0` at tag `v0.2.0`
Release range: `v0.2.0..v0.2.1`

## Why 0.2.1

0.2.1 is a focused post-0.2.0 Signal Packet export release.

It does not change crawler behavior, backend contracts, extension storage
shape, or `DLENS_SIGNAL_PACKET_VERSION`. The shipped packet/read-model changes
are additive and safe for older readers to ignore.

## What shipped

### Signal Packet contract

- HTML exports now use compact density and mark
  `data-signal-packet-density="compact"`.
- HTML signal provenance strips include source/capture/item metadata when
  present.
- `DLensSignalEvidence.citedInReadingRefs` maps evidence refs back to the
  SignalReading cache keys that cited them.
- `DLensSignalReadingBundle` exposes `latestFiled` and `supersededFiled`, while
  keeping `reading.latest` as the visible reading body source.
- `DLensSignalPacket.source` exposes source URL provenance:
  `urlSource`, `pageUrlSource`, `pageUrlFallbackSource`, and
  `canonicalTargetUrlSource`.

### HTML export affordances

- Cited evidence rows show which readings cite the evidence.
- The Reading panel shows filed lineage:
  `最新 filed <cacheKey>` and, when present, `另有 N 個歷史 filed`.
- Superseded filed reading bodies are intentionally not rendered in the HTML
  surface.
- URL fallback provenance is visible in the same `signal-meta` strip, for
  example `pageUrl via url fallback (reading.sourcePacket.postUrl)`.

### Product popup mirror

- Product Reading Review provenance rows now mirror the packet provenance strip:
  `來源`, `capture`, and `item`.
- The data comes through `ProductSignalViewModel`, so the View still does not
  read storage or call browser APIs directly.

## Contract notes

- `DLENS_SIGNAL_PACKET_VERSION` remains `v3`.
- No storage migration.
- No crawler selector or parser change.
- No backend tag in this release.
- PR #52 (`codex/backend-jobs-c0-audit`) remains a separate draft docs PR and is
  not part of 0.2.1.
