## N1 — 戀愛被當成市場 / 算術問題

### Story shape
*Setup*：發聲者面對擇偶 / 感情。*Tension*：他們不用情感語言處理，而用市場、評分、收入、性價比的語言處理。*Outcome*：戀愛被轉譯成可計算的定位 / 篩選問題——OP 或建構這套算術（S2）、或抗議它（S3）、或被它困住（S7）。

### Core evidence
- **`S2.OP`** 是純粹的市場理論：把女性按「7-9 分」評級，男人按「top 5%」分層，引入「競爭池」「挑剔模式」「機會率同性價比」——戀愛被完整 de-romanticize 成 supply-demand optimization。
- **`S3.OP`** 抗議這套算術：批判 app 自介「好似見工咁」列「有車有樓」，反諷「不如直接寫埋月薪幾多」。OP 反對市場化，但**反對的方式仍是引用市場詞彙**（月薪 / 供完樓）。
- **`S7.OP`** 被算術困住：用收入軌跡（月入 6 萬 / 萬 9→10 幾萬）為三任男友編號，最後「要認命」。`S7.R2` 直接指認這套算術是意識形態（「用收入衡量…受資本影響」）。

### Lexicon hook（P2）
P2 §1.1 市場 / 評分 / 算帳詞群橫跨 S2/S3/S7，與 work 算帳詞群同源。

### Boundary / inversion
**`S6.OP` 是純反例**：偷偷接男友收工，全篇沒有一個算術 / 評分 / 收入詞，只有 reciprocity 與情緒（開心 / 嘩嘩聲）。S6 證明戀愛市場化不是唯一語言——但它是 9 篇中**唯一**完全跳出市場框架的訊號。

### 不處理的（移交 P5）
- 沒有 signal 主張「戀愛不應該被市場化」並提出替代框架——S3 抗議但仍用市場詞，S6 示範但不立論
- 沒有男性視角的市場敘事（評分系統全由女性 OP 建構 / 抗議）

## N2 — 價值對撞：理想主義 OP vs 市場現實主義 reader

### Story shape
*Setup*：OP 提出一個關於戀愛 / 擇偶的 value position（理想主義或道德判斷）。*Tension*：reader **不接 OP 的立場，從相反 value position 反駁**。*Outcome*：top reply 區出現真正的 disagreement——這是 work 幾乎沒有的 discourse 形態。

### Core evidence
- **`S3`**：OP 持 romantic-essentialist（想知興趣不是身家），`S3.R1` + `S3.R2` **都為功利化辯護**（防詐 / 大勢所趨）——reader 站市場現實主義反對 OP。
- **`S4`**：OP thesis（好對象唔喺 app），`S4.R1` 樂觀反駁（我同老公就係 app 識）+ `S4.R2` cynical 反駁（app 上的好條件者係掠食者）——**OP 被兩個相反方向同時拆**。
- **`S7`**：OP 用收入衡量男友，`S7.R2` 直接質疑這個 value frame 本身（受資本影響 / 香港女性通病）——reader 不回應 OP 處境，而是 attack OP 的衡量標準。

### Lexicon hook（P2）
P2 §2.2：love 的 reader-coined 集中在 reframe OP 的心態 / 價值觀（心態決定行為 / 受資本影響），而非補完省略——reader 在 love 傾向**校正 OP 的認知框架**。

### Boundary / inversion
- **`S6`** 是反例：reader **一致 endorse**（文言文祝福 / 對方有回應就係幸福），沒有任何 disagreement——正面甜蜜敘事不引發對撞。
- **`S5`** 是反例：reader 只「留友」（bookmark），不表態——dark humor 不引發對撞。
- **`S2`** 想對撞但**看不到**：OP 自評「極度具爭議性」並預期反對，但 audience reply 在 preview 中為 0——對撞不可得（evidence gap）。

### 不處理的（移交 P5）
- value clash 全部停在 individual value（你太務實 / 你太天真 / 你受資本影響），**沒有升級到「我們社會的擇偶價值應該如何」的集體層**
- 沒有 reader 改變立場 / 達成共識的 evidence——對撞呈現但不收斂

## N3 — 平台作為敵人 + 退場衝動

### Story shape
*Setup*：發聲者在交友 app 上經歷挫敗（已讀不回、奇葩、掠食者、演算法操弄）。*Tension*：挫敗被歸因於平台本身（app 的設計 / app 上的人）。*Outcome*：回應是「離開平台」——退 app、del app、勸退——而非改變策略或要求平台改善。

### Core evidence
- **`S1.OP`** 問已讀不回者的心態，`S1.R1`「已退 apps」+ `S1.R2`「唔好玩」——reader 答「離開」。
- **`S9.OP`** 詳述 app 奇葩後「好灰想 del app」——退場衝動。
- **`S2.OP`** 把平台升級為敵人（「演算法存在惡意，App 設計係為咗留住用戶」）——OP 的策略結論是「靚女應該回到現實空間」，即退出 app。

### Lexicon hook（P2）
P2 §1.2 exit 詞（已退 apps / 想 del app）+ §1.4 退場詞群。

### Boundary / inversion
- **`S6`** 反例：關係發生在 app 之外（既有男友），平台不是敵人——甚至沒提 app。
- **`S4`** 是 N3 的極端版：OP 不只想退場，而是主張**好對象根本不在 app 上**——把平台從「敵人」升級為「錯誤場域」。但 `S4.R1` 用「我同老公係 app 識」反駁這個前提。

### 不處理的（移交 P5）
- 沒有 signal 要求平台改善 / 監管演算法——退場是唯一回應，「修平台」缺席（對照 work：work 也只有個體回應，沒有集體 / 制度回應）
- 退場後去哪裏（現實空間如何運作）只有 S2 提了（回到現實顯示優勢），其餘無

## Inversion case：S6 — 被擁抱的正面敘事

`S6`（偷偷接男友收工）在三條敘事都是反例，值得單獨記。它是 9 篇中唯一純正面、無算術、無對撞、無平台挫敗的訊號，且 **reader 一致 endorse 並升華**（`S6.R1` 文言文「執子之手與子偕老」、`S6.R2`「對方有回應就係幸福」），OP likes 2300 為全 topic 最高。

S6 的存在說明：love 的語言生態**容許正面敘事被擁抱**——不像 work 唯一正面敘事（創業 S6）被 self-undercut + reader 諷刺壓制。但 S6 內部有一個 ironic 對位：OP handle `no_responsesss`（無回應）vs `S6.R2`「對方有回應就係幸福」——**正面敘事被放在「回應稀缺」的背景中閱讀**，它的甜蜜恰恰因為周遭（S1/S9）的無回應而顯得珍貴。

## Cross-cutting register：戲謔 / 身體化 / 認命

穿插各敘事的修辭層（無獨立 story shape）：
- dark humor：`S5`（墊 m 巾）、`S8`（玩十條 JJ）——把脆弱 / 慾望降壓成可笑
- 認命 / 放棄：`S7`（要認命）、`S9`（好灰想 del app）——以 resignation 收尾
- 身體化：`S5`（生理期）、`S8`（JJ）、`S9`（腹肌 / 二頭肌）——情感 / 慾望被身體詞承載

與 work 戲謔 register 同功能（讓不可說的可說），但 love 多了**身體 / 性**的維度，且 love 的 resignation（認命）比 work 更直接（work 多用 self-rhetorical question 開放收尾）。
