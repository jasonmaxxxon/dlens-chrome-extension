## Stage Contract

### Purpose
觀察 reader 對 OP 做了什麼動作。Codex 建議的 checklist：接住 / 打回 / 升級 / 變行動建議——但**看到才寫，沒看到就不寫**。可以擴展也可以縮減，不強制把每篇套進固定 4 種。

### Output
- 觀察到的反應型態（自由命名，不限數量）
- 每種型態的 evidence + 引述
- Reader 反應 NOT 觀察到的（簡記，作為 P5 hook）
- Cross-reaction 觀察（哪類 reader 動作對應哪類 OP）

### What this pass does NOT do
- 不重寫 narrative cluster（P3 完成）
- 不寫 speaker / position 缺席（P5）
- 不為反應型態打分 / 排優先級 / 打 enum
- 不引用 Codex northstar

### Discipline
- 只記錄 P0 commentsPreview 裡實際可見的 reply。S15 queued 無 reply 資料，本 pass 不收
- 引用必須掛 `S#.R#`
- 區分 organic reader reply vs OP 自我回覆 / pinned comment（後者不算 audience reaction）
- 不從 likes 數推「整個議題的 audience 共識」——top 3 reply 只能代表 top 3，不代表全部 reader

### 不算 audience reaction 的 evidence（必須排除）
- `S6.R1`：OP `shingi_shiratori` 自我回覆，是 OP amplification 不是 reader reaction
- `S8.R1`：pinned comment 平台 placeholder（無作者欄），不確定是否 organic top reply
- `S8.R2`：OP `just_for_justise` 自我轉貼 R1，是 OP amplification

排除後可用的 audience reply：14 篇 × 平均 2 條 ≈ 27 條（扣 S15 的 0）。

## 1. 觀察到的反應型態

### 1.1 接住但不選邊（emotional permission without prescription）

Reader 對 OP 的恐懼 / 自卑 / 焦慮提供 emotional reframe，但**不主張 OP 應該留下還是離開**——把抉擇還給 OP，自己只負責同行。

**Evidence:**
- `S2.R1` manddddys（64 likes）：「唔好驚做錯野，有時老員工都會做錯，假如接受唔到新員工做錯，間公司容錯率都幾低，做錯野之後能學習番就好啦」——用「容錯率」reframe 公司而非 OP，但沒建議 OP 該做什麼
- `S2.R2` master.atse.10（43 likes）：「橫掂都想辭職，難得嚟到嚟個場，帶住一個反正預咗死既心態，盡量學佢哋既嘢」——給 nihilism 當 emotional 免疫力，仍不選邊

**特徵：** Reader 在 OP self-discount（「係咪我太弱」）之後，不反向 boost OP（沒人說「你唔弱」），也不反對 OP 的恐懼是否合理。**等於同意 self-discount，但給陪伴**。

### 1.2 協作擴寫 OP 自鑄的 vocabulary

OP 創造了一個身份標籤 / rebellion script，reader 接力擴展技術細節，沒有人質疑這個 script 是否合理。

**Evidence:**
- `S3.OP` 自鑄「薪水小偷行動升級」
- `S3.R1` lau02051984（5 likes）：「飲多啲水加速排毒兼運動，連屋企電費都慳埋，因為唔駛煲水，一舉三得」——over-justification 喜劇延伸
- `S3.R2` keungson_mo（5 likes）：「去廁所記得去耐啲」——直接升級為 operational protocol

**特徵：** Reader 動作不是 endorsement、不是同情、是**接力寫腳本**。OP 提供 vocabulary frame，reader 擴展 protocol。這個動作只在 S3 出現於完整形態。S11 的「全民整水喉」punchline 引出 `S11.R1`「睇緊點考水電A牌」屬於弱版本——擴寫但只是跟風 meme，沒到 protocol 層。

### 1.3 Reader 補完 OP 的省略

OP 極簡（一句到四句），reader 用 reader-coined 詞 + 具體痛點補出 OP 沒寫的論述。

**Evidence:**
- `S13.OP` 一句 + 一個數字 → `S13.R1` mfmf8611（309 likes）reader-coined「高級興趣班」+ `S13.R2` e_is_missing_again（243 likes）以 employer 身份補出「依家見到 master degree 嘅 candidate 通常都唔會考慮」這個 OP 沒寫的招聘 bias
- `S14.OP` 四句白描 → `S14.R1` dennis._.n（281 likes）reader-coined「Permanent head Damage」+ `S14.R2` liviakwan121（150 likes）補具體痛點「對住同一樣嘢幾年，仲要唔一定做到 positive result」

**特徵：** 兩個訊號都是極簡 OP、reader-coined 詞 likes 達到三位數、reader 用比 OP 更精煉的 vocabulary 接管論述。P2 §2.2 已標記「reader-coined 詞集中在學歷主題」——P4 把這個觀察具體化為一種 reader 動作。**OP 越省略，reader 越補位**這個 pattern 在這兩篇成立。

### 1.4 同方向升級為結構分析 / 二階推論

OP 提出某個結構性命題或諷刺，reader 升級為更精細的論述——對照分析、feedback loop、政策對比。**不反駁，同方向 escalate**。

**Evidence:**
- `S9.R1` mkm2bby（114 likes）：519 字結構分析——對照新加坡外勞政策、地盤鋁模生產力（「人哋一個頂你四個人」）、外勞工資 1/3-1/4、政府監管缺位。OP 用粗口 + hashtag 帶情緒；R1 用案例 + 對比把它變成 policy 論述
- `S11.R2` sebastianium（32 likes）：feedback loop 推論「個個被炒嘅都走去整水喉，競爭激烈...冇水喉俾你整...整水喉都冇得做」——把 OP 列舉的事件接成因果鏈

**特徵：** Reader 不抗辯，而是用 second-order 思考把 OP 的點延伸。`S9.R1` 是 horizontal escalation（地理 / 制度對照），`S11.R2` 是 vertical escalation（時間 / 因果推演）。**結構性議題的 reader 比情境性議題的 reader 更會升級**——這呼應 N4「結構性歸因但無 prescriptive」的另一面：reader 願意 analyze 不願意 prescribe。

### 1.5 Credential-based disqualification

Reader 自報 expert / insider 身份（醫生 / employer / 業界），用 credential 否決 OP 提出的 option 或補充 OP 沒看見的事實。

**Evidence:**
- `S12.R1` edwardyftse（2900 likes，全 audit 最高 reply）：「千祈唔好讀護士，現在已經飽和，政府準備招聘更多內地護士來港。(我係醫生）」——三件事：禁止 + 事實 + credential
- `S13.R2` e_is_missing_again（243 likes）：以招聘者身份揭露 anti-master degree bias「依家見到 master degree 嘅 candidate 通常都唔會考慮」
- `S5.R1` bizmarket.hk（42 likes）：給 415 字 franchise 經濟拆解，自報業界身份「有野唔明再 dm 我問」——但 R1 是 conditional endorsement 不是純 disqualify

**特徵：** Credential 動作出現在 OP 處於「求問是否可走某條路」狀態時（S5、S12、S13）。**Reader 用 expertise 完成出路的關閉**（這也是 N2「出路同步自證偽」的主要機制）。三條 reply 的 likes 體量（2900 / 243 / 42）跟 credential 的權威性 ranking 一致：醫生 > 招聘者 > 業界 advisor。

### 1.6 集體 topic-shift reframe

Reader 三條 top replies 一致把 OP 想討論的議題重新 frame 成另一件事——OP 想討論 X，reader 把它讀成 Y。

**Evidence（只有 1 篇強 case）：**
- `S7.OP` 想討論「同一大學 / 學院仍有結構性區別」的階級議題；reader 三條一致：
  - `S7.R1` unknownduck_duck（1600 likes）：「你講嘢唔分場合，上唔到大枱」——讀成社交失態
  - `S7.R2` winniefatpoon__（964 likes）：「冇咩必要主動講出嚟...你係唔恰當嘅場合自爆」——讀成 over-disclosure
  - `S7.R3` jacko.488（673 likes）：「你太放大自己個出身，現實根本冇咁多觀眾」——讀成自我中心

**特徵：** 三條 reply 共 3237 likes、方向完全一致、論點全部不接 OP 的 frame 而是改 frame。OP 的「階級結構」議題在 top 3 reply 內完全沒被處理。這是 15 篇中唯一的「集體 reframe」case——其他訊號 reader 大致接 OP frame（即使反方向也是同 frame）。

### 1.7 跟風 meme tail（戲謔 register 延續但不升級）

Reader 接 OP 的 punchline 或 vocabulary 但只是 meme 跟風，沒擴展、沒升級、沒給 protocol。

**Evidence:**
- `S1.R2` diuneion9ac（1 likes）：「say on god?」——跟風 meme，不接 OP 議題
- `S11.R1` hoyin.eth（38 likes）：「睇緊點考水電A牌」——順 OP「全民整水喉」punchline 跟風

**特徵：** 跟風出現於戲謔 register 強的訊號，但只在 top reply 第二位、likes 低。**Reader 想參與戲謔 register 但沒能力 / 沒意願擴寫**——這是 cross-cutting register（P3 標記）的弱反應形態。

### 1.8 Reader 內部分裂並列 advice

Top replies 給互相矛盾的 survival logic，OP 自己沒選邊，讀者面對兩種策略並列。

**Evidence（只有 1 篇強 case）：**
- `S10.OP` 困境：17k vs 1000 蚊請食 vs 不能跳槽
- `S10.R1` diu_lookwhat7ar（2200 likes）：「得 17k 仲要請，叫佢地食屎啦」——拒絕規則
- `S10.R2` vialaviida（1400 likes）：「呢啲文化嘅公司啲人都好撚小氣記仇...就請咗大家飲嘢算 9 數」——接受規則

**特徵：** R1 + R2 都是 actionable advice，但策略相反（attack the rule vs survive the rule）。雙方 likes 都遠超 OP（2200 + 1400 vs 788）。**這篇的 audience 不是不選邊，是兩派同時被高度認同**——讀者群本身是分裂的。15 篇中唯一這樣的 case。

### 1.9 Individual-level actionable prescription

Reader 給具體可執行建議，但範圍嚴格限定在 individual 層級（如果你要做 X，這樣做），不主張集體 / 制度層的 prescription。

**Evidence:**
- `S5.R1` bizmarket.hk（42 likes）末尾：「要做既最好搵 50k 租樓下，同埋頂手鋪再翻新。慳裝修同出牌」——具體 actionable，限於 individual franchise 操作
- `S10.R2` vialaviida（1400 likes）：「短期內唔想轉工，就請咗大家飲嘢算 9 數」——pragmatic 妥協，限於 individual 試用期生存
- `S10.R1` diu_lookwhat7ar（2200 likes）：「叫佢地食屎啦」——是 attitude prescription 不是 procedure prescription，但仍在 individual 層

**特徵：** P3 N4 已標記「結構性命名 + 沒 prescription」——P4 補充：**prescription 不是不存在，是 scope 嚴格限定在 individual 層**。沒有 reply 主張「行業協會應該」「政府應該」「同行應該一起」。`S9.R1` 519 字結構分析也沒給 prescription，只給 diagnosis。

### 1.10 質疑來源 / 諷刺 attribution

Reader 不質疑 OP 主張的內容（事實 / 論點），質疑這個主張的**來源 / 起點**。

**Evidence:**
- `S6.R2` kokorotrading（2 likes）：「加埋老豆老母比嘅幾千萬碎銀 終於辛辛苦苦做到以上嘅野」——不否定 OP 列舉的資產，反而懷疑這些資產的 attribution（繼承而非創業）

**特徵：** 1 個訊號、1 條 reply、likes 低（2）。但在邏輯動作層面與其他反應型態完全不同——其他反應處理 OP 的 argument，這條處理 OP 的 self-presentation。對應 P3 N3「經營體面」敘事的反向 audit。這是孤例但結構特殊。

### 1.11 Topic drift to unrelated content

Reader 整體漂移到非 OP frame 的話題。

**Evidence:**
- `S4.OP` frame「外國返工 vs 香港 share screen」+ Spoiler 圖（不可得）
- `S4.R1` kaka_inca（59 likes）：「大便唔臭原因係咪你鼻塞？」
- `S4.R2` sophiawong0418（21 likes）：「網絡上已經冇你在意既人啦？😂💩」

**特徵：** 唯一明確的 topic drift case。reader 完全不討論 OP 的 frame（工作場景 / 地理 / share-screen 文化），全部聚焦在 visual 內容（廁所 / 排泄）。Visual 不可得導致無法判定 drift 是 reader 自發還是 OP 視覺內容引導——這個 ambiguity 必須在 Validator 標記。

### 1.12 Advice-prohibition violation

OP 明說「不要建議」, reader 仍然給 reassurance / advice。

**Evidence:**
- `S1.OP`「唔係求咩解決辦法 只係想🤧下」
- `S1.R1` mprispa003（1 likes）：「90尾你都係 GenZ, 好快融入啦」——既 reassure 也重新分類，違反 OP 兩個 prohibition：別 advise + 別重新分類

**特徵：** 1 個訊號、1 條 reply、likes 極低（1）。但是少數記錄到 OP 與 reader 在「對話協議」層級不對齊的 case——OP 想設定的對話規則被 reader 無視。

## 2. 反應 NOT 觀察到（移交 P5）

下列 reader 動作在 15 篇 top 3 reply 內**沒出現**。這只代表 top 3 沒有，不代表整個 comment thread 沒有；但 top 3 是讀者投票結果，「沒出現在 top 3」本身是訊號。

### 2.1 沒有為 OP 反向 boost 的 reply

當 OP self-discount（「係咪我太弱」`S2`、「本身自己又唔係叻」`S10`、「唔敢同老婆講」`S12`），**沒有 reader 反向主張「你冇問題 / 你唔弱 / 你應該講」**。Reader 接住 self-discount 但不挑戰它。`S2.R1` 改 frame 到公司容錯率、`S2.R2` 用 nihilism 包圍，但都不反駁 OP 對自己的否定判斷。

### 2.2 沒有 collective / institutional prescription

Individual-level prescription 出現（§1.9），但**沒有「行業 / 政府 / 工會 / 同行 / 媒體 / 學校應該做 X」的 reply**——即使在 N4 強訊號（S9 反外勞、S11 AI 裁員、S8 薪俸稅）的 top 3 reply 內也沒有。Reader 願意做 diagnosis（§1.4 同方向升級），但 prescription scope 不擴展到 individual 層以上。

### 2.3 沒有 reader 為 escape ramp 辯護

P3 N2「出路同步自證偽」的 reader 動作清一色是 disqualification（§1.5）或內部分裂（§1.8）。**沒有 reader 主張某條 escape ramp 確實可行**——`S5.R1` 是 conditional endorsement（如果你要做就這樣做）不是 unconditional defense。沒有人說「讀護士其實可以」、「轉行可以」、「跳槽會更好」。

### 2.4 沒有 reader 質疑 OP 是否誠實

除 `S6.R2`（§1.10）質疑 attribution（資金來源），**沒有 reader 質疑其他 OP 的 self-report 是否真實**——沒有 reader 說「你個 17k 係咪講大」、「你話飽和係咪過度悲觀」、「你母親係咪真係咁講」。OP 的 self-report 在這個 reader 群裡有極高的 prima facie credibility。

### 2.5 沒有跨訊號 reader（cross-thread participation）

15 篇 commentsPreview 的 top reply authors **沒有重複**——每個 reply 的作者只出現在那一個訊號內。這代表這個 audit 的 reader pool 沒有重複出現的「議題參與者」，每個訊號的 audience 是獨立的。這也限制了「audience reaction」是否能視為穩定的群體模式——可能只是各個訊號的偶然採樣。

## 3. Cross-reaction 觀察

### 3.1 反應型態與 OP 類型的對應

| OP 類型 | 主要 reader 動作 |
|---------|------------------|
| Minimalist OP（S13, S14） | 補完省略 + reader-coined reframe（§1.3） |
| 求問 escape ramp 的 OP（S5, S12, S13） | Credential disqualification（§1.5） |
| 結構性命題 / 時評 OP（S9, S11） | 同方向升級為結構分析 / feedback loop（§1.4） |
| 元披露 / 表演 OP（S7） | 集體 topic-shift reframe（§1.6） |
| Self-discount + 困境 OP（S2, S10, S12） | 接住不選邊 或 內部分裂（§1.1, §1.8） |
| 戲謔 / 諷刺 OP（S11, S14） | 跟風 meme tail 或 reader-coined（§1.7, §1.3） |
| 元披露 + 列舉成就 OP（S6） | 質疑來源（§1.10）+ OP 自我 amplification |
| Topic frame 模糊 + visual OP（S4） | Topic drift（§1.11） |
| Advice-prohibition OP（S1） | Advice-prohibition violation（§1.12） |
| 自鑄 rebellion vocabulary OP（S3） | 協作擴寫（§1.2） |

**最強對應：** Minimalist OP → reader 補完省略 + reader-coined reframe。S13 + S14 兩篇 reader-coined 詞 likes 達三位數，呼應 P2 §2.2 「reader-coined 詞集中於學歷主題」——這條對應在 lexicon + audience 兩層都成立。

**最弱對應：** 沒有單一強對應的 OP 類型，是 self-discount OPs——這類 OP 同時觸發接住、內部分裂、credential disqualification 三種不同反應，audience response 散開。

### 3.2 高 reply likes 集中分佈

Top reply likes ≥ 1000 的訊號（reader 動作獲得高度認同）：

- `S7.R1` 1600、`S7.R2` 964、`S7.R3` 673——topic-shift reframe（§1.6），三條全在 top
- `S10.R1` 2200、`S10.R2` 1400——內部分裂並列 advice（§1.8）
- `S12.R1` 2900——credential disqualification（§1.5）
- `S13.R1` 309、`S13.R2` 243——補完省略 + credential（§1.3 + §1.5）
- `S14.R1` 281——補完省略（§1.3）

**沒有任何高 likes reply 屬於「接住不選邊」「跟風 meme」「協作擴寫 vocabulary」這幾類**。Top likes 集中在動作型 reply（reframe / disqualify / actionable / coined 詞）。**Reader 群偏好可量化、有定論的 reply——猶豫 / 陪伴 / 戲謔不會獲得高 likes**。

### 3.3 OP-likes vs Top-reply-likes 的對比

- Reply > OP（top reply 比 OP 更被認同）：S10、S12、S5（三篇）
- OP >> Reply（OP 主導）：S14（4000 vs 281）、S13（1900 vs 309）、S11（400 vs 38）、S2（235 vs 64）
- 接近：S9（268 vs 114）、S8（193 vs 38）
- 反向（OP 體量低 + reply 也低）：S1、S3、S4、S6

**Pattern：** Reply 超過 OP 的情況集中在「OP 求問 + reader 用 credential 否決出路」（S10、S12）或「OP 提問 + reader 給 actionable」（S5）。**Credential 動作的市場價值高於提問本身**——讀者用 likes 表達他們認可 disqualification 的權威性。

### 3.4 戲謔 register 在 reader 端的 fate

P3 標記戲謔 register 是 cross-cutting。P4 補上：

- OP 戲謔 → reader 跟風 meme（弱版本，§1.7）：S1.R2、S11.R1
- OP 戲謔 → reader reader-coined 延續（強版本，§1.3）：S14.R1
- OP 戲謔 → reader 協作擴寫成 protocol（最強版本，§1.2）：S3.R1/R2
- OP 戲謔 → reader 切換 register 給 actionable：沒有觀察到

**Reader 不會從戲謔切回 serious register**——一旦 OP 進入戲謔 register，reader 留在那個 register。這呼應 P3 cross-cutting register 觀察：戲謔 register 不能升級為 prescriptive。在 audience 端這個 register lock 同樣成立。
