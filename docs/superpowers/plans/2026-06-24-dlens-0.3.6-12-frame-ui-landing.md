# DLens 0.3.6 12-Frame UI Landing Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Land `docs/mockups/2026-06-23-dlens-720x860-12-frames.html` into the live popup while preserving the full 12-frame journey and the current View/ViewModel boundary.

**Architecture:** Treat the HTML as an implementation reference, not a second design system. Build a small set of shared evidence/decision primitives in React, then adopt them in Product, Topic, and PR surfaces one mode at a time. Do not change backend contracts, storage shape, classifier logic, or command targets unless a failing test proves the current ViewModel cannot support the UI.

**Tech Stack:** React 19, TypeScript, WXT MV3, existing inline token system in `src/ui/tokens.ts`, Node `tsx` tests, DOM/static-markup characterization tests.

## Frame Mapping

Keep all 12 frames in the product journey:

1. Product Intake: existing Product saved-signals/inbox route; lighter triage.
2. Product Classification: existing classification route; keep bucket view and representative quote.
3. Product Action Brief CORE: existing `actionable-filter`; preserve macro strip `信號重試 / 噪音不符 / 資料不足 / 保留觀察`.
4. Product Packet Export: existing Signal Packet / Reading Review export section; make export quieter.
5. Topic List: existing `TopicsListView`.
6. Topic Detail: existing topic audit overview + themes + narrative lanes.
7. Topic Drill-in CORE: existing `SignalDrawer` / source row drill-in; original text first, AI annotation second.
8. PR Setup: existing `CampaignEditor`; make criteria setup feel like AI has drafted and user reviews.
9. PR Evidence Ledger: existing `EvidenceLedger`; one-line match indicator, not six noisy chips.
10. PR Criteria Health CORE: new PR health section inside `PrEvidenceView`, derived from existing campaign rows and criteria.
11. PR Export Preview: existing `CsvPreview` / summary export, quieter and non-admin.
12. Shared Evidence Drill-in Appendix: developer/reference-only; do not expose as a product route.

## Non-Goals

- Do not add a new design-system markdown file.
- Do not add a second mode switch inside Product/Topic/PR pages.
- Do not render the mockup's caption bars such as `PRODUCT MODE · 行動簡報` inside the live popup.
- Do not add fake analytics like `target 80`, `14 captured`, or invented campaign IDs.
- Do not replace real Product macro labels with generic chips.
- Do not make Frame 12 a product page.
- Do not bypass ViewModel / typed command target / storage seam / boundary guards.

## Rollout Notes

- This is a user-visible UI release. If merging to `main`, keep the existing release hygiene: sync `package.json`, `package-lock.json`, `wxt.config.ts`, and `src/ui/version.ts` only if the version changes.
- `package.json` already reports `0.3.6` at the time this plan was written; verify whether `wxt.config.ts`, `src/ui/version.ts`, and built manifest are already aligned before bumping again.
- The safest PR shape is two commits minimum:
  1. shared primitives + tests
  2. mode adoption + tests
- If the diff grows beyond reviewable size, split after Task 4. Product/Topic/PR are independent once shared primitives are in place.
